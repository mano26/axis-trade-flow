1 — Workflow Overview

Every trade follows the same sequence. Completing each step in order ensures the ticket and cards are accurate before printing.

  1. Enter the trade string on the Orders page and submit. The system parses the string and creates a confirmed order record.
  2. Record a fill once the trade is done. Enter the filled quantity (may be partial).
  3. Save leg prices — enter the price for each leg. The system validates that leg prices reconcile to the stated package premium.
  4. Save counterparties — enter the filling broker, counterparty, and bracket for each lot allocation.
  5. Print cards and ticket — only available once all counterparties are allocated.
For partial fills, repeat steps 2–4 for each partial. Each fill gets its own counterparty allocation. Cards and tickets always reflect the complete order.

2 — Trade String Syntax

The general structure is:

<contract> <strike(s)> <C/P> <strategy> [ratio] <price_format>

Contract Codes

Use standard CME Globex codes for SOFR options. Examples:

SFRH6 SFRM6 SFRU6 SFRZ6 SFRH27 SFRM7 SFRU7 SFRZ7 0QH6 0QM6 2QZ6

Short-dated pack codes (0QN6, 2QM6, etc.) are recognised automatically.

Strikes

List one or more strikes as decimals after the contract code. Order does not matter — the system sorts them correctly for each strategy.

SFRH6 96.25 CS 4/500 <- one strike per leg inferred SFRH6 96.00 96.25 CS 4/500 <- explicit strikes SFRH6 95.75 96.00 96.25 C FLY 2/200

Option Type

Use C or CALL for calls, P or PUT for puts. For strategies that imply a type (CS, PS, BFLYC, BFLYP, etc.) the type token may be omitted.

4 — Price Format & Direction

The price/quantity token tells the system both the package premium and the direction of the trade.

Buy (Debit)

price/qty e.g. 4/500 (pay 4 ticks for 500 lots)

The number before the slash is the price in ticks. The number after is the quantity.

Sell (Credit)

qty@price e.g. 500@4 (collect 4 ticks on 500 lots)

Spaces around @ and / are accepted: 500 @ 4 and 500@4 both work.

Prices are in ticks. The system converts to decimal internally (4 ticks = 0.04 points). Tick size for SOFR options is 0.25 of a basis point (0.0025), so 4 ticks = 0.01 points.

5 — Advanced Modifiers

Ratio Spreads

Add a ratio token like 1X2, 1X3, or 1X2X1 to create unequal-leg structures. The base quantity from the price format is multiplied by each ratio number.

SFRU6 96.3125 96.1875 P 1X2 7.5/4000 <- buy 4000 @ 96.3125, sell 8000 @ 96.1875 SFRH6 96.00 96.25 96.50 C FLY 1X2X1 2/200

CVD (Covered / Delta Hedge)

Appending CVD <futures_price> D <delta%> adds a futures leg to the trade at the specified delta.

SFRH6 96.00 C 3/500 CVD 96.38 D 40 <- buy 500 calls, sell 200 futures (40% delta) SFRH6 96.00 C 3/500 CVD 96.38 D 40 (+) <- force futures to buy side SFRH6 96.00 C 3/500 CVD 96.38(-) D 40 <- force futures price to sell

VS — Opposing Segments

VS splits a trade into two segments going in opposite directions — the standard calendar/spread-of-spread separator.

SFRM6 96.00 C VS SFRU6 96.00 C 4/500 <- buy M6, sell U6 call

By default the first segment takes the direction of the price format (buy for /, sell for @). To specify which segment is the buy/sell without reordering, add a contract code in parentheses at the end of the string. The named contract takes the price-format direction; the other flips.

SFRM6 96.375 ^ VS SFRU6 96.50 ^ 16/1000 <- buys M6, sells U6 (default) SFRM6 96.375 ^ VS SFRU6 96.50 ^ 16/1000 (SFRU6) <- sells M6, buys U6 SFRM6 96.375 ^ VS SFRU6 96.50 ^ 1000@16 (SFRU6) <- buys M6, sells U6
The parenthetical hint must be an exact contract code (e.g. SFRU6, SFRH7, 0QM6). Anything else — a note, a number, a strike — is stripped and ignored as usual.

WITH — Same-Direction Segments

WITH splits a trade into two or more segments going in the same direction. Use it when the two segments have different strategies or different contracts.

SFRM6 96.75 ^ WITH SFRZ6 96.00 C stupid 25/1000 <- buy straddle AND buy call SFRM6 96.00 96.25 PS WITH SFRM6 97.50 98.00 CS stupid 10/200

STUPID / STRIP

Modifiers that indicate all segments go the same direction (no direction flip).

  • STUPID — same-direction 2-segment trade
  • STRIP — same-direction 3+ segment trade (typically multiple contracts, same strike)
SFRU6 96.00 96.25 C stupid 4/500 <- buy both calls SFRM7 SFRU7 SFRZ7 SFRH8 99.00 C strip 45/750 <- buy all 4 contracts

Bracket Wrapper [ ]

Wrap multiple comma-separated segments in square brackets with a shared price/quantity at the end. Each segment inside uses its own contract and strategy but shares the package price.

[SFRH6 C 96.00, SFRM6 P 95.50] 4/500

6 — Quarter-Tick Check

When a trade price falls on a 0.25-tick increment (e.g. 2.25 ticks, 4.75 ticks) rather than a half-tick or whole-tick, the system pauses and asks for confirmation before submitting.

A dialog will appear:

  • OK — the contract month is quarter-tick eligible. Submit as a parsed trade.
  • Cancel — submit as a generic trade so you can enter legs manually.
Quarter-tick pricing is only valid for eligible expiry months. If you are unsure, click Cancel and enter as generic.

7 — Generic Trade Mode

For complex or non-standard trades that the parser cannot handle, check the Generic Trade box before submitting. The trade string is stored as a reference but the system does not attempt to parse it into legs.

After creating a generic order:

  1. Go to the order detail page. The legs table will be empty and fully editable.
  2. Enter each leg manually: B/S, volume, market, contract, expiry, strike, C/P.
  3. Click Save Legs. You can add or remove rows before saving.
  4. Then proceed with Record Fill → Save Prices → Save Counterparties as normal.
The package premium from the trade string is used to validate prices on a generic order. Make sure the total net dollar value of all legs (qty × price) divided by the total order quantity equals the stated premium.

Split fills / averaged prices: You can enter the same side as two rows at different prices to represent an average fill. For example, two buy rows at 500 lots each for a 1000-lot order. Price validation will average them against the stated premium correctly.

8 — Recording Fills

Single Fill

On the order detail page, enter the fill quantity in the Record Fill section and click Record Fill. For a complete fill enter the remaining quantity (pre-filled by default).

Partial Fills

Enter a quantity less than the total to record a partial fill. The order status becomes PARTIAL FILL. Repeat as needed — each partial gets its own set of prices and counterparties. The balance remains open for additional fills.

Save Prices

Enter the price for each leg in the Confirmation Legs table. When there are multiple pending fills, prices are saved to the oldest unallocated fill first. The system validates that prices reconcile to the package premium before saving.

Save Counterparties

For each fill enter the filling broker, counterparty symbol, bracket, and quantity. Quantities must sum to the fill quantity. Once saved the fill status becomes ALLOCATED.

If you record a second partial before allocating the first, both pending fills will appear with separate entry forms. Work from top to bottom — allocate the earliest fill first.

9 — Cards & Tickets

Cards and tickets are available from the order detail page and the order log once all fills are fully allocated.

Cards

One card is printed per leg per bracket. Each card shows the quantity, contract/month, strike, premium, counterparty, and bracket. For multi-leg trades the bracket letter is circled and suffixed with 6 to indicate a spread.

Quantities on the card are scaled by the leg's ratio relative to the base lot size. For a 1×2 spread where the counterparty allocated 1000 lots at the base leg, the 2× leg card will show 2000.

Ticket

One ticket per order. Shows all legs on both buy and sell sides, plus timestamps (IN, FILL, MOD, OUT) all in Chicago time. All brackets used across all fills are circled in the bracket row at the bottom.

Use your browser's Print function (Ctrl+P) on the cards or ticket page. The print stylesheet removes the navigation bar and formats each card to 3.5×5.5 inches, each ticket to 8×5.5 inches.

10 — Amending a Trade

After a fill is fully allocated you can still correct it. On the order detail page, find the allocated fill in the Counterparties section and click Edit.

The amendment form lets you change:

  • Leg prices (re-validated against the package premium)
  • Filling broker, counterparty, and bracket for each allocation
  • House and account

Save Amendment commits both price and counterparty changes in a single transaction. The order status updates to AMENDED and a full before/after record is written to the audit trail.

To amend the trade structure itself (strikes, strategy, direction) use the Modify Order button. This re-parses a new trade string and rebuilds the legs.

11 — Order Log

The Order Log shows all orders across any date range. Use the filters to search by:

  • Date range — defaults to today
  • Ticket number — type a full or partial number
  • Strategy — type the strategy code (e.g. cs, bflyc, ptree)
  • House — partial match
  • Account — partial match
  • Search — searches the raw trade string (e.g. type SFRK6 to find all trades on that contract)

Click any row to open the order detail. Cards and tickets are accessible directly from the log for filled orders.
